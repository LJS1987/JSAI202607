# DIY 자율주행 프로젝트 로드맵

일반 차량에 카메라·레이더·라이다와 조향/가감속 제어를 추가하여,
주행하면서 학습하고 현실 세계의 3D 맵을 구축하는 자율주행 시스템을 만드는 것이 목표.

---

## 0. 시작하기 전에 반드시 알아야 할 것 (법규·안전)

- **공도(일반 도로)에서 조향·브레이크를 개조 장치로 조작하는 것은 승인 없이는 불법이다.**
  - 한국: 자동차관리법상 조향·제동장치 변경은 **튜닝 승인 대상**이며, 자율주행 시험운행은
    국토교통부의 **자율주행자동차 임시운행허가**가 필요하다.
  - 무승인 개조 차량은 보험 적용이 거부될 수 있고, 사고 시 전적으로 본인 책임이 된다.
- 따라서 개발 순서는 반드시 다음을 따른다:
  1. **인지(Perception)와 데이터 수집** — 차를 전혀 개조하지 않고 합법적으로 가능
  2. **시뮬레이션 및 축소 모형(1/10 스케일)** — 제어 알고리즘 검증
  3. **실차 제어는 사유지·폐쇄 코스에서만**, 항상 즉시 개입 가능한 상태로
- 브레이크·조향 액추에이터는 어떤 경우에도 **운전자가 물리적으로 이길 수 있는(overridable)**
  구조여야 하고, 킬 스위치(전원 차단)를 반드시 둔다.

---

## 1. 전체 아키텍처 (모듈형 자율주행 스택)

```
[센서]                [인지]              [측위/맵]           [판단/계획]        [제어]
카메라 ──┐        객체 검출/추적 ─┐      SLAM / 3D 맵    ─┐   경로 계획      횡방향(조향)
레이더 ──┼─ 융합 ─ 차선/신호 인식 ─┼─→   HD맵 + RTK-GPS ─┼─→ 행동 결정  ─→  종방향(가감속)
라이다 ──┘        깊이 추정      ─┘      로컬라이제이션  ─┘   궤적 생성       (CAN 또는 액추에이터)
IMU/GPS
```

- 테슬라 FSD는 End-to-End(카메라→조작 직접 학습) 방식이지만, 개인 프로젝트는
  **모듈형(위 구조)** 으로 시작해야 디버깅이 가능하다. E2E/모방학습은 데이터가 쌓인 뒤 실험.

---

## 2. 단계별 로드맵

### Phase 1 — 기초 학습 + 데이터 로거 (차량 무개조, 지금 바로 시작)

**목표: 내 차를 "데이터 수집 차량"으로 만든다.**

- 학습: Python, **ROS 2**, OpenCV, PyTorch 기초, 선형대수/좌표변환
- 하드웨어 (약 100~200만원):
  - 컴퓨터: NVIDIA **Jetson Orin Nano/NX** 또는 중고 노트북 + RTX GPU
  - 카메라: 전방 광각 1~2대 (글로벌 셔터 권장, 예: Arducam/oCam) — 초기엔 웹캠도 가능
  - GPS+IMU: u-blox ZED-F9P (**RTK-GPS**, cm급) + 9축 IMU
- 할 일:
  - ROS 2로 카메라/GPS/IMU 동기화 로깅 (rosbag)
  - 카메라 캘리브레이션 (내부/외부 파라미터)
  - YOLO 등으로 차량/보행자 검출, 차선 인식 파이프라인 구동

### Phase 2 — 차량 인터페이스 분석 (CAN 버스)

**목표: 차가 이미 갖고 있는 센서·제어 신호를 읽는다.**

- OBD-II 포트에 CAN 인터페이스 연결 (comma **panda**, CANable, Kvaser 등)
- `candump`, SavvyCAN, comma **cabana**로 조향각, 휠속도, 페달, 순정 레이더 신호 리버싱
- **핵심 분기점**: 내 차에 전동 파워스티어링(EPS) + 스마트 크루즈(ACC)가 있는가?
  - **있다** → 물리 액추에이터 없이 **CAN 메시지로 조향/가감속 명령 가능** (openpilot 방식).
    이것이 가장 현실적이고 안전한 "개조 키트" 경로다.
  - **없다** → 모터로 핸들/페달을 물리적으로 움직여야 함. 난이도·위험도가 크게 오르므로
    차량 교체 또는 폐쇄코스 전용 프로젝트로 전환을 권장.
- **지름길**: 내 차가 [openpilot 지원 차종](https://comma.ai/vehicles)이면 **comma 3X** 장착이
  사실상 유일한 검증된 상용 개조 키트다. 먼저 이걸로 레벨2를 경험하고,
  openpilot 소스(오픈소스)를 분석하는 것이 최고의 교재가 된다.

### Phase 3 — 축소 모형으로 제어 검증 (실차보다 먼저!)

**목표: 조향/가감속 알고리즘을 안전하게 검증한다.**

- **F1TENTH**(1/10 RC카 + 라이다 + Jetson) 또는 DonkeyCar / JetRacer
- 실차와 동일한 스택(ROS 2, SLAM, 경로계획, PID/MPC 제어)을 그대로 돌려본다
- 병행: **CARLA 시뮬레이터**에서 인지→계획→제어 전체 루프 테스트, 모방학습/강화학습 실험

### Phase 4 — 라이다 도입 + 3D 매핑

**목표: "현실 세계 3D 가상 맵"을 만든다.**

- 라이다: **Livox Mid-360**(약 100만원대, 입문 최적) 또는 중고 Ouster/Velodyne
- LiDAR-Inertial SLAM: **FAST-LIO2**, LIO-SAM → 주행하며 포인트클라우드 맵 생성
- RTK-GPS로 지리좌표 정합(georeferencing) → 재방문 시 맵 위에서 자기위치 추정(localization)
- 시각화 고도화: 포인트클라우드 → 메시화, 또는 **3D Gaussian Splatting**으로
  사진급 3D 맵 재구성 (테슬라식 "가상 세계" 구현에 해당)

### Phase 5 — 주행 데이터 학습 (데이터 엔진)

**목표: 주행할수록 똑똑해지는 구조를 만든다.**

- 매 주행을 rosbag으로 축적 → 자동 라벨링(오토라벨: 대형 모델로 오프라인 라벨 생성)
- 내 주행(조향각·페달 로그)을 정답으로 하는 **모방학습(Imitation Learning)** 실험
- 시뮬레이터(CARLA)에 내가 만든 3D 맵을 넣어 corner case 재현·학습

### Phase 6 — 실차 제어 통합 (폐쇄 코스 한정)

- Phase 2에서 확보한 CAN 제어 (또는 승인된 액추에이터)로 횡방향(조향) 제어부터
- 제어기: PID → **MPC(Model Predictive Control)**
- 안전장치: 운전자 토크 감지 시 즉시 해제, 킬 스위치, 속도 제한, 워치독
- 공도 확장은 임시운행허가 등 법적 절차를 갖춘 뒤에만

---

## 3. 추천 시작 순서 (요약)

1. **이번 달**: ROS 2 + Python 학습, Jetson/GPU 확보, 차에 카메라+GPS 로거 설치
2. **1~3개월**: CAN 분석으로 내 차의 제어 가능성 판정, openpilot 지원 여부 확인
3. **3~6개월**: F1TENTH 스케일카 + CARLA로 제어 스택 완성, 라이다 구입
4. **6~12개월**: FAST-LIO2로 동네 3D 맵 구축, 데이터 엔진 구축
5. **그 이후**: 폐쇄 코스 실차 제어, 모방학습 실험
