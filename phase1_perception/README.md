# Phase 1 — 인지(Perception) 첫 실습

휴대폰/블랙박스 주행 영상에 **YOLO 객체 검출 + 차선 인식**을 돌려보는 첫 실습.

## 준비물

- 주행 영상 파일 1개 (mp4 권장)
  - 휴대폰을 차량 거치대에 두고 전방을 찍거나, 블랙박스 SD카드에서 영상을 복사
- Python 3.10 이상
- (선택) NVIDIA GPU — 없어도 동작하지만 처리 속도가 느림

## 설치 (최초 1회)

```bash
# 가상환경 생성 및 활성화 (권장)
python -m venv venv
source venv/bin/activate        # Windows는: venv\Scripts\activate

# 라이브러리 설치
pip install -r requirements.txt
```

## 실행

```bash
python detect.py 주행영상.mp4
```

- 처음 실행하면 YOLO 모델 파일(`yolo11n.pt`, 약 6MB)이 자동 다운로드된다.
- 완료되면 `주행영상_result.mp4` 파일이 생성된다. 열어보면:
  - **색깔 박스**: YOLO가 검출한 차량/보행자/신호등 등 (신뢰도 점수 포함)
  - **초록 선/영역**: 차선 인식 결과와 주행 가능 영역

옵션:

| 명령 | 설명 |
|---|---|
| `python detect.py 영상.mp4 --show` | 처리하면서 실시간 화면 표시 (`q`로 종료) |
| `python detect.py 영상.mp4 --model yolo11s.pt` | 더 정확한(느린) 모델 사용 |
| `python detect.py 영상.mp4 --no-lane` | 차선 인식 끄고 YOLO만 |

## 결과를 보고 관찰할 것 (이게 진짜 공부)

1. **잘 되는 상황**: 낮, 직선 도로, 선명한 차선 → 왜 잘 되는지
2. **깨지는 상황**: 곡선 도로, 야간, 비, 역광, 차선이 지워진 곳, 앞차가 차선을 가릴 때
3. 깨지는 장면들이 바로 자율주행의 어려움이자, 딥러닝 기반 차선 인식과
   센서 융합(레이더/라이다)이 필요한 이유다.

## 코드 구조

- `detect.py`
  - YOLO 검출: 사전학습된 `yolo11n` 모델 사용 — **우리가 학습시킬 필요 없음**
  - 차선 인식: 고전 컴퓨터비전 (흑백 변환 → Canny 엣지 → 관심영역 마스크 → 허프 변환 → 좌/우 차선 분류)
- 카메라 장착 위치가 다르면 `region_of_interest()`의 사다리꼴 비율을 조정할 것

## 다음 단계

1. 차선 인식이 깨지는 장면 모으기 → 고전 CV의 한계 체감
2. 딥러닝 차선 인식(예: Ultra-Fast-Lane-Detection, CLRNet)으로 교체 실험
3. 카메라 캘리브레이션 배우기 → 검출된 객체까지의 거리 추정으로 확장
